# kanji-pay (幹事ペイ)

幹事の集金ストレスをゼロにする、チップ機能付きスマート割り勘決済アプリケーション。

---

## 1. サービス概要 (Service Overview)
`kanji-pay` は、飲み会やイベントの幹事が「集金・割り勘」を最もシンプルかつスピーディに行うためのWebアプリケーションです。
面倒な個人間送金（QRコードの提示や電話番号入力）を排除し、シェアされたリンクを開くだけでスマートに決済が完了します。
さらに、幹事への感謝の気持ちを伝える「チップ機能」を搭載し、幹事が報われる世界を目指します。

---

## 2. 解決する課題 (Problem Solved)
* **幹事の負担**: 割り勘計算、集金状況の管理、未払い者への催促が精神的・時間的に大きな負担であること。
* **参加者の面倒**: 個人間の送金時に、QRコードの読み取りや電話番号・口座番号の入力が必要で手間がかかること。
* **幹事の「働き損」**: 準備やお店選び、進行をすべて担ってくれた幹事に対して、感謝を形にする手段がないこと。

---

## 3. コア機能 & ユーザーフロー (Core Features & UX)

### ① 幹事フロー (主催者側)
1. イベント終了時、幹事が `kanji-pay` にアクセス。
2. **「合計金額」** と **「参加人数」** を入力。
3. アプリが自動で割り勘金額を計算し、**「一専用の決済リンク」** を発行。
4. 幹事はこのリンクをLINEやSlackなどのグループチャットに「皆さんここから支払いをお願いします！」とシェアするだけ。

### ② 参加者フロー (支払者側)
1. 共有されたリンクをスマホで開く。
2. 画面に自動計算された **「一人あたりの割り勘金額」** が表示される。
3. **「チップ選択エリア」** で、幹事への感謝として追加で送るチップを選択（任意）。
   * 選択肢例: `なし` / `+¥100` / `+¥200` / `+¥500` / `+¥1,000` / `+¥2,000`
4. Apple Pay、Google Pay、またはクレジットカード（Stripe決済）で瞬時に決済完了。

---

## 4. 手数料・マネタイズモデル (Monetization)
本サービスでは決済ごとに **合計 5.0%** の手数料を差し引いて幹事へ送金されます。

* **Stripe決済手数料**: `3.4%` (決済プラットフォーム利用料)
* **プラットフォーム利益 (マージン)**: `1.6%` (運営・開発費)

> **💡 ビジネスモデルのメリット**
> 幹事は「手間の削減」と「チップ獲得のチャンス」を得られるため、5.0%の手数料は十分に許容範囲となります。また、チップが発生した場合は手数料分を容易に回収・プラスにできる仕組みになっています。

---

## 5. システム構成 (System Architecture)
本アプリケーションは、既存のバックエンドAPIを有効活用するマルチクライアント構成をとります。

* **フロントエンド (`kanji-pay`)**: React / HTML5 / Vanilla CSS などによるモダンなSPA（シングルページアプリケーション）。モバイルファーストの直感的で美しいUI/UXを実装。
* **バックエンド (`shukin-api`)**: 共通の集金APIサーバー。決済リンク生成、決済状態の管理、Stripeとの通信を担当。

## 6. 今後のロードマップ (Roadmap)

### Phase 1: MVP（最小限のプロダクト）
* 合計金額・人数入力からのStripe決済リンク生成
* モバイル向け決済画面の実装
* 任意チップ選択機能（¥100〜¥2,000）
* Stripeによるテスト決済連携

### Phase 2: 幹事用ダッシュボード
* 幹事アカウント登録機能
* 過去のイベント集金履歴一覧
* 誰が支払いを完了したかのリアルタイムステータス追跡

### Phase 3: 幹事支援機能の拡張（集金以外の機能）
* 飲み会の日程調整・出欠確認機能（調整さんライクな機能）
* 店舗検索・候補リスト共有機能
* イベント当日用リマインダーや出欠管理
